# docs/spec-match-screen.md — 麻雀対戦画面仕様

4人麻雀を**実際にプレイする**ための対戦画面仕様を定義する。
本画面は結果を表示・入力するだけの画面ではなく、対戦開始から終局まで人間と CPU が実際に対戦できる画面とする。
ゲーム進行・CPU AI のドメイン仕様は [docs/spec-phase2.md](./spec-phase2.md) を参照。

## スコープ

| 含む | 含まない |
|---|---|
| 対戦開始時の初期表示 | 点数計算エンジンの実装変更 |
| 局情報・供託・本場の表示 | オンライン対戦・通信同期 |
| 4人分のプレイヤー情報と持ち点表示 | 牌譜保存・観戦機能 |
| 牌山生成・配牌・手牌の表示と操作 | 三人麻雀 |
| ツモ・打牌・鳴き・リーチ・和了・パスの操作 | |
| CPU の自動対局(自動ツモ・自動打牌・自動和了) | |
| 和了・流局時の結果表示と点数反映 | |

## 対戦開始設定

対戦開始前に、一般的な麻雀アプリで必要になる以下の設定を選択できる仕様とする。

### プレイヤー設定

4席それぞれについて、以下を設定できる。

| 項目 | 内容 |
|---|---|
| 操作種別 | `人間` / `CPU` |
| 表示名 | 画面に表示するプレイヤー名 |
| CPUレベル | CPUの場合のみ選択する強さ |

CPUレベルは初期仕様では以下の3段階とする。各レベルの具体的な行動要件は [docs/spec-phase2.md](./spec-phase2.md) の「CPU 対局 AI」を参照。

| レベル | 想定挙動 |
|---|---|
| 入門 | 和了・安全牌よりも単純な牌効率を優先する初心者向け |
| 標準 | 牌効率、役、守備をバランスよく判断する通常対戦向け |
| 上級 | 押し引き、危険牌、役価値をより重視する高難度向け |

### ルール設定

対戦開始前に、以下のルールを選択できる。

| 項目 | 選択肢 | 初期値 |
|---|---|---|
| 対戦形式 | `東風戦` / `半荘戦` | `半荘戦` |
| 持ち点 | 25,000点開始 / 30,000点開始 | 25,000点開始 |
| 返し点 | 30,000点 / なし | 30,000点 |
| 喰いタン | あり / なし | あり |
| 後付け | あり / なし | あり |
| 赤ドラ | あり / なし | あり |
| CPU自動和了 | あり / なし | あり |

## 対戦の進行

「対戦開始」を押すと [docs/spec-phase2.md](./spec-phase2.md) のゲーム進行エンジンで対局を開始し、以下を満たすこと。

- 配牌が行われ、人間プレイヤーの手牌が画面に表示される。
- 手番が自動で進行する。CPU の手番では CPU が自動でツモ・打牌・鳴き・和了を行う。
- 人間の手番では、そのとき可能なアクション(打牌、チー、ポン、カン、リーチ、ツモ、ロン、パス)だけが操作可能になる。
- 他家の打牌に対して人間が鳴き・ロンできる場合は、宣言またはパスを選択できる。
- 和了・流局で局が終了すると結果を表示し、確定すると点数反映のうえ次局へ進む。
- 東風戦/半荘戦の終了条件を満たすと最終順位を表示する。

### 対戦中の補助表示

対戦画面には、一般的な麻雀アプリで確認できる以下の補助情報を表示する。

- 残り山枚数
- ドラ表示牌
- 捨て牌河(各プレイヤーの捨て牌)
- 鳴き面子
- リーチ状態
- 現在の手番
- 操作可能なアクション(打牌、チー、ポン、カン、リーチ、ツモ、ロン、パス)

## 初期表示

対戦画面を開いた直後は、標準的な4人半荘戦の開始状態を表示する。

| 項目 | 初期値 |
|---|---:|
| 局 | 東一局 |
| 本場 | 0本場 |
| 供託 | 0本 |
| 東家の持ち点 | 25,000点 |
| 南家の持ち点 | 25,000点 |
| 西家の持ち点 | 25,000点 |
| 北家の持ち点 | 25,000点 |

## 表示項目

### 卓全体

- 現在の局を表示する。
- 本場数を表示する。
- 供託リーチ棒の本数を表示する。
- 親がどのプレイヤーか分かるように表示する。

### プレイヤー

各プレイヤーについて、以下を表示する。

- 風: `東` / `南` / `西` / `北`
- 表示名
- 持ち点
- 親・子の状態

## 和了・流局結果の表示と反映

和了・流局は対局中の操作(ツモ・ロン宣言、山切れ)から**自動的に**発生する。手動で結果を入力して反映する方式は採用しない。

和了時は点数計算エンジン(Phase 1)を公開API経由で呼び出し、以下を結果表示に含める。

| 項目 | 内容 |
|---|---|
| 和了者 | 和了したプレイヤー |
| 放銃者 | ロンの場合のみ、放銃したプレイヤー |
| 和了種別 | `ツモ` / `ロン` |
| 成立役 | 役名と翻数の一覧 |
| 翻数 | 点数計算結果の翻数 |
| 符 | 点数計算結果の符 |
| 点数 | 本場・供託を含む点数移動 |
| 親継続 | 連荘か輪荘か |

流局時はテンパイ/ノーテンの別とノーテン罰符の点数移動を表示する。

## 点数反映

結果を確定したとき、画面上の対戦状況は以下の規則で更新する。

- 和了者の持ち点を増やす。
- 支払い者の持ち点を減らす。
- 供託がある場合、和了者へ供託点を加算し、供託本数を0に戻す。
- 親が和了した場合は親を継続し、本場数を1増やす。
- 子が和了した場合は次局へ進め、親を次のプレイヤーへ移す。
- 子が和了した場合、本場数は0に戻す。
- 流局時はノーテン罰符を移動し、親のテンパイ有無に応じて連荘/輪荘とする(詳細は [docs/spec-phase2.md](./spec-phase2.md))。

## 追加仕様の反映方針

別ブランチなどで対戦画面に関する仕様が追加された場合は、本書の「対戦開始設定」「対戦の進行」「表示項目」「和了・流局結果の表示と反映」「点数反映」のいずれかへ分類して追記する。
ゲーム進行・CPU AI に関する追加仕様は `docs/spec-phase2.md` へ、点数計算エンジンの入出力や役・符計算に関する追加仕様は `docs/spec-phase1.md` へ追記する。
